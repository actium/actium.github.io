# Ускорение операций ввода/вывода

## Синхронизация потоков

Языки программирования C и C++ предоставляют стандартные потоки для выполнения операций ввода/вывода.
По умолчанию работа потоков синхронизируется, что позволяет использовать их совместно в одной программе.
На практике это означает, что потоки C++ не буферизуются, и каждая операция ввода/вывода в потоке C++
применяется непосредственно к буферу соответствующего потока C. Синхронизацию можно отключить вызовом
`std::ios_base::sync_with_stdio(false)`.

!!! warning ""
    При отключении синхронизации потокам C++ будет позволено использовать собственные независимые буферы,
    что может привести к непредсказуемым последствиям при вызове функций типа `printf` и `scanf` вперемешку с
    операторами ввода/вывода.

!!! important ""
    Стандарт C++ не гарантирует сохранность данных, если синхронизация отключается после выполнения операций
    ввода/вывода (буферизованные данные могут быть уничтожены).

## Связывание потоков

По умолчанию `std::cin` привязан к `std::cout`. Это гарантирует, что буфер потока вывода будет автоматически
сброшен перед операцией ввода. Например, при выполнении строк кода

```cpp
  std::cout << "Enter name:";
  std::cin >> name;
```

можно ожидать, что подсказка `Enter name:` будет выведена до того, как программа запросит у пользователя ввод
данных. Поток ввода можно отвязать от потока вывода вызовом `std::cin.tie(nullptr)`.

!!! warning ""
    При отвязывании возможна ситуация, когда чтение данных уже началось, а подсказка "застряла" в буфере, т.е.
    еще не выведена (практически вывод производится либо по требованию, либо при переполнении буфера).

!!! tip ""
    Если потоки ввода и вывода не связаны и нужно вывести данные перед запросом ввода, то буфер вывода можно
    сбросить вручную (`std::cout << std::flush` или `std::cout << std::endl`).

!!! note ""
    Некоторые участники соревнований по незнанию используют вызов `std::cout.tie(nullptr)`, который не имеет
    никакого смысла, поскольку по умолчанию поток вывода ни к чему не привязан.

## Практические рекомендации

При решении большинства задач имеет смысл использовать такую строку кода:

``` cpp
  std::cin.tie(nullptr)->sync_with_stdio(false);
```

Она отключает синхронизацию потоков C и C++ и отвязывает `std::cin` от `std::cout`, что заметно ускоряет
ввод и вывод данных.

Для дополнительного ускорения вывода можно сократить частоту сброса буфера, если вместо `std::endl`
использовать `'\n'`.

Иногда встречаются интерактивные задачи, при решении которых лучше оставить потоки связанными, чтобы не
прибегать к ручному сбросу буфера, и лишь отключить синхронизацию с stdio:

``` cpp
  std::ios_base::sync_with_stdio(false);
```
